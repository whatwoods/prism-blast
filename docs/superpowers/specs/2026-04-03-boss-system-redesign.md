# Boss System Redesign Spec

## Overview

Prism Blast Boss 系统全面重构。将当前简陋的八面体 + 旋转方块 Boss 替换为基于粒子系统的"能量风暴"Boss，引入双模态循环战斗节奏，增加 Boss HUD，强化打击反馈。

**核心目标**：
- 视觉从实体几何体升级为粒子能量形态
- 战斗节奏从单调的"打护盾→打核心"变为紧张的远近交替循环
- 增加 Boss 血条和完整的打击反馈系统
- 代码架构封装化，预留多 Boss 扩展能力

**约束**：
- 保持单 HTML 文件架构
- 使用现有 THREE.js 引擎，不引入新依赖
- 粒子数 200-300，确保性能流畅
- 先做好一种 Boss，架构可扩展

---

## 1. Boss 视觉设计 — 能量风暴体

### 核心构成

Boss 不再是实体几何体，而是由粒子系统构成的能量生命体。

**基础粒子云**（200-300 个粒子）：
- `THREE.Points` + `BufferGeometry`，每个粒子是发光光点
- 粒子围绕中心按椭球分布，Simplex noise 驱动湍流运动
- 基础色调：青紫渐变（`#00f0ff` → `#8800ff`）

**能量核心**（弱点）：
- 中心小型发光球体（`SphereGeometry` + `MeshBasicMaterial`，additive blending）
- 持续脉动（scale 0.8-1.2 呼吸），是粒子云中最亮的点
- 被击中时闪白 + 脉动加剧

### 两种形态的视觉差异

| | 风暴态（远程） | 聚合态（逼近） |
|---|---|---|
| 粒子分布 | 散开，半径大，快速旋转 | 收缩凝聚，半径小，密实 |
| 粒子运动 | 湍流明显，混沌感 | 向前流动，有攻击方向感 |
| 核心可见度 | 粒子间隙可见，可攻击 | 被粒子包裹，需打散粒子层 |
| 色调 | 偏青蓝冷色 | 偏紫红暖色，越近越红 |
| 后处理 | 正常 bloom | bloom 加强 + 色差加重 |

### 受伤视觉反馈

- 命中：粒子短暂向外爆散后回收，核心闪白
- HP 降低：粒子运动越不稳定，颜色偏移越大，出现能量"漏电"效果（随机方向小粒子喷射）
- 濒死：整体闪烁不定，粒子频繁脱落

## 2. 战斗循环 — 双模态攻防节奏

### 整体流程

```
登场动画 → [风暴态 ↔ 聚合态] 循环 N 次 → 狂暴态 → 死亡
```

### 登场阶段（约 2 秒）

- 隧道远处出现光点，迅速膨胀为粒子云
- 屏幕震动 + 色差 + 警告文字
- 顶部 Boss 血条滑入，显示名称"维度风暴"
- 登场期间 Boss 无敌，玩家不受攻击

### 风暴态（远程弹幕期，计时器驱动，持续 7 秒）

**转换条件**：持续 7 秒后自动转入聚合态。

**Boss 行为**：
- 停留在隧道远端（z ≈ -20），粒子散开旋转
- 持续发射投射物，频率 = 基础频率 * 循环轮次衰减系数：
  - 第1轮：每 0.8 秒一波，每波 3 发
  - 第2轮：每 0.6 秒一波，每波 4 发
  - 第3轮：每 0.5 秒一波，每波 5 发
  - 公式：间隔 = max(0.3, baseInterval - cycleCount * 0.15)，每波数量 = 2 + cycleCount
- 投射物模式交替：扇形散射 → 螺旋 → 随机散布
- 能量核心在粒子间隙暴露，玩家可边拦截弹幕边攻击核心

**玩家策略**：拦截投射物保命的同时，寻找间隙射击核心。拦截优先级判断是核心技能考验。

### 聚合态（逼近威胁期，事件驱动）

**转换信号**：粒子突然向中心收缩（0.5 秒），颜色转红，音效警告。

**Boss 行为**：
- 粒子凝聚成密实体，快速向玩家逼近
- 移动速度：8 + 循环轮次 * 2 单位/秒
- 逼近时前方持续掉落碎片投射物（低频但体积大）
- 核心被粒子层包裹，必须先打散外层粒子才能伤害核心
- 外层粒子有独立 HP 池，打光后核心短暂暴露 2 秒

**威胁机制**：
- Boss 推进到 z ≈ -5 时，玩家扣 2 命 + Boss 强制后退
- 玩家输出足够伤害可让 Boss 减速/停止/击退
- 每次命中粒子层使 Boss 移动速度临时降低 10%（0.3 秒内恢复）

**转回风暴态**（满足任一）：粒子层 HP 归零，或 Boss 推进到 z ≈ -5 触发惩罚后强制后退。转回后粒子层 HP 按新轮次数值重置。

### 循环加压

| 循环轮次 | 弹幕间隔(秒) | 每波数量 | 聚合态速度(单位/秒) | 粒子层 HP |
|---------|------------|---------|-------------------|---------|
| 1 | base | 3 | 8+lv*1.5 | 8+lv*3 |
| 2 | base-0.15 | 4 | 10+lv*1.5 | 12+lv*3 |
| 3 | base-0.30 | 5 | 12+lv*1.5 | 15+lv*3 |
| 4 | base-0.40 | 6 | 14+lv*1.5 | 18+lv*3 |
| 5 | base-0.40 | 6 | 14+lv*1.5 | 18+lv*3 |

其中 base = max(0.4, 0.8 - bossLevel * 0.05)。弹幕间隔下限 0.3 秒。

### 狂暴态

**触发条件**（OR 关系，满足任一即触发）：
- Boss HP 降至 20% 以下
- 循环轮次达到 5 轮

**行为**：
- 不再分模态，两种威胁同时存在
- Boss 缓慢逼近（速度 4 单位/秒）+ 持续高密度弹幕（每 0.4 秒一波，每波 5 发）
- 粒子极度不稳定，核心持续暴露（无需打粒子层）
- 红白交替闪烁
- 考验玩家极限压力下的最后输出

### 死亡动画（约 2 秒）

- Boss 定住，粒子向核心剧烈收缩
- 核心过载爆炸，粒子向四面八方喷射消散
- 隧道被冲击波照亮（全屏白闪 + bloom 最大化）
- 碎片变为金色能量点消散

## 3. Boss HUD 与反馈系统

### 血条 UI

**位置**：屏幕顶部居中，战斗开始时从上方滑入。

**组成**：
- Boss 名称："维度风暴 Lv.X"，小号字，血条上方
- 主血条：宽度约屏幕 60%，高度 8px
  - 底色：深灰半透明
  - 填充：渐变色（青蓝 100% → 紫 50% → 红 20%）
  - 受击时血条闪白，减少部分用浅色"拖尾"延迟 0.5 秒消失
- 阶段指示：血条下方小文字（"风暴态" / "聚合态" / "狂暴"）
- 狂暴阈值：血条 20% 位置有一条细竖线标记

**粒子层血条**（聚合态专用）：
- 仅聚合态显示，位于主血条下方
- 较短较细，颜色偏紫
- 表示外层粒子护盾剩余 HP

### 打击反馈

| 事件 | 视觉反馈 | 屏幕效果 |
|------|---------|---------|
| 命中核心 | 核心闪白 + 粒子爆散 | 轻微屏震 + hit stop 30ms |
| 命中粒子层 | 命中点粒子碎裂飞散 | 无 |
| 打穿粒子层 | 大范围粒子爆散 + 核心暴露闪光 | 屏震 + 色差 + 提示"核心暴露！" |
| Boss 被击退 | Boss 整体后移 + 粒子拖尾 | 轻微屏震 |
| 进入狂暴 | 全体粒子红白爆闪 | 强屏震 + 色差 + 警告"狂暴！" |
| Boss 死亡 | 收缩→爆炸→金色消散 | 全屏白闪 + 最大 bloom |

### 伤害数字

- 命中核心时在命中点弹出伤害数字，向上飘散消失
- 白色小字，连续命中时变大变金色
- 用 canvas 2D overlay 实现

## 4. 数值设计与奖励系统

### Boss 基础数值（bossLevel = wave / 10）

| 属性 | 公式 | Lv1 | Lv3 | Lv5 |
|------|------|-----|-----|-----|
| 总 HP | 30 + bossLevel * 15 | 45 | 75 | 105 |
| 粒子层 HP | 8 + bossLevel * 3 | 11 | 17 | 23 |
| 风暴态弹幕基础频率 | max(0.4, 0.8 - bossLevel * 0.05) 秒 | 0.75s | 0.65s | 0.55s |
| 聚合态移动速度 | 8 + bossLevel * 1.5 单位/秒 | 9.5 | 12.5 | 15.5 |
| 狂暴阈值 | 总 HP * 20% | 9 | 15 | 21 |

### 伤害规则

- 子弹命中核心：1 点伤害
- 子弹命中粒子层：1 点粒子层伤害（不影响主 HP）
- Boss 投射物未拦截：玩家扣 1 命
- 聚合态 Boss 推进到 z ≈ -5：玩家扣 2 命 + Boss 强制后退

### 奖励

**击败奖励**：
- 分数：`8000 * bossLevel`
- 全额回满生命值

**永久强化**（三种轮替）：
- bossLevel % 3 === 1：额外子弹 `extraBalls++`，"永久强化：每发 +1 球"
- bossLevel % 3 === 2：射速提升 `fireRateBonus += 0.02`，"永久强化：射速提升"
- bossLevel % 3 === 0：子弹穿透 `piercingBonus++`，"永久强化：穿透 +1"

穿透效果：子弹命中障碍物后不消失，可继续击中后方目标，穿透次数 = piercingBonus。注意：穿透机制需要修改现有的子弹-障碍物碰撞逻辑（非 Boss 专属代码），在碰撞时检查 piercingBonus 决定子弹是否销毁。

### 难度曲线

- Wave 10（首个 Boss）作为教学关：弹幕稀疏、逼近慢
- 循环轮次上限 5 轮
- 5 轮后强制进入狂暴态收束战斗

## 5. 架构设计

### 重构策略

保持单 HTML 文件架构。文件内部用封装的 Boss 对象组织代码。

### Boss 对象接口

`createBoss(bossLevel)` 工厂函数返回：

```javascript
{
  // 状态
  hp, maxHp, phase, cycleCount,
  particleShieldHp, maxParticleShieldHp,

  // 三维对象
  group,           // THREE.Group 根节点
  particles,       // 粒子系统 (THREE.Points)
  core,            // 核心球体

  // 方法
  update(dt),      // 主更新，内部按 phase 分发
  handleHit(hitPos), // 被击中，返回 {damaged, amount, type}
  getHitTargets(), // 碰撞目标列表 [{pos, radius, type}]
  dispose(),       // 清理资源

  // HUD
  getHudData(),    // {hp, maxHp, phase, name, shieldHp, maxShieldHp}
}
```

### 主循环集成

```javascript
if (bossActive) {
  boss.update(dt);
  checkBossCollisions(boss);
  updateBossHud(boss.getHudData());
}
```

### 阶段内部组织

Boss `update(dt)` 内部按 phase 分发到独立函数：
- `updateEntrance(dt)` — 登场动画
- `updateStorm(dt)` — 风暴态逻辑
- `updateConverge(dt)` — 聚合态逻辑
- `updateFrenzy(dt)` — 狂暴态逻辑
- `updateDeath(dt)` — 死亡动画

阶段转换条件和动画封装在 Boss 内部。

### 扩展性

未来添加新 Boss 类型：创建新的 `createBossTypeB(bossLevel)` 工厂函数，返回相同接口（`update`、`handleHit`、`getHitTargets`、`getHudData`、`dispose`），主循环零修改。

### 粒子系统技术方案

- `THREE.Points` + `BufferGeometry`
- position / color / size 用 `Float32BufferAttribute`
- 每帧更新 position attribute（`needsUpdate = true`）
- Simplex noise 内联轻量实现（约 50 行）
- 200-300 粒子，性能可控

## 6. 需要删除的旧代码

替换以下现有函数：
- `spawnBoss()` → 替换为 `createBoss()`
- `updateBoss()` → 替换为 `boss.update()`
- `hitBoss()` → 替换为 `checkBossCollisions(boss)`
- `bossAttack()` → 内聚到 Boss 对象内部
- `destroyBoss()` → 替换为 `boss.dispose()` + 奖励逻辑

保留的外部逻辑：
- Boss 触发条件（wave % 10 === 0）
- 奖励发放（分数、永久强化）
- `bossActive` 全局标记
